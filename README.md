# Professional Portfolio

Open `index.html` in a browser to view the portfolio.

To publish with GitHub Pages, upload every file in this folder to the root of the repository, then set Pages to publish from the `main` branch and `/ (root)` folder.

Before submitting, replace the placeholder biography, contact links, and artifact links with your real AIML-500 coursework materials. Each artifact entry already follows the required structure:

- Title
- Objective
- Process
- Tools
- Artifact-specific value proposition

Reflections should be submitted to Brightspace separately, not placed on the public portfolio.
